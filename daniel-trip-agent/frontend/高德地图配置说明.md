# 高德地图配置说明

## 问题现象
前端地图显示灰色格子，无法正常显示地图瓦片。

## 原因
高德地图 JS API 2.0 需要配置**安全密钥（securityJsCode）**才能正常使用。

## 解决步骤

### 1. 登录高德地图开放平台
访问：https://console.amap.com/

### 2. 进入应用管理
- 点击"应用管理" → "我的应用"
- 找到你的应用（如果没有则创建一个新应用）

### 3. 获取Web端（JS API）密钥
- 在应用中添加或查看 "Web端（JS API）" 密钥
- 记录下 **Key** 和 **安全密钥（securityJsCode）**

### 4. 配置环境变量
编辑文件：`frontend/.env`

```env
# 高德地图Web API Key
VITE_AMAP_WEB_KEY="你的Key"

# 高德地图安全密钥
VITE_AMAP_SECURITY_JS_CODE="你的安全密钥"
```

### 5. 重启前端服务
```bash
cd frontend
npm run dev
```

## 注意事项

1. **Key 和 securityJsCode 必须匹配**
   - 两个值都来自同一个应用的同一个密钥配置

2. **Web服务和Web端（JS API）是不同的**
   - 后端使用的是 "Web服务" Key（用于API调用）
   - 前端使用的是 "Web端（JS API）" Key（用于地图显示）
   - 这两个Key是不同的！

3. **检查域名白名单**
   - 在高德平台配置中，确保添加了 `localhost` 和 `127.0.0.1` 到域名白名单
   - 如果部署到服务器，需要添加服务器域名

## 验证配置是否成功

1. 打开浏览器开发者工具（F12）
2. 访问行程规划结果页面
3. 查看控制台（Console）：
   - 如果配置正确：会显示 "地图加载成功"
   - 如果配置错误：会显示错误信息，如 "INVALID_USER_SCODE" 或 "USERKEY_PLAT_NOMATCH"

## 常见错误

### 错误1: INVALID_USER_SCODE
**原因**：安全密钥（securityJsCode）配置错误
**解决**：检查 VITE_AMAP_SECURITY_JS_CODE 是否正确

### 错误2: USERKEY_PLAT_NOMATCH
**原因**：Key 和 securityJsCode 不匹配
**解决**：确保两个值来自同一个应用的同一个密钥

### 错误3: INVALID_USER_KEY
**原因**：Key 配置错误或已过期
**解决**：检查 VITE_AMAP_WEB_KEY 是否正确

### 错误4: 域名不在白名单中
**原因**：当前访问域名未在高德平台配置
**解决**：在高德平台添加域名白名单
