在 Agentic 应用中，状态通常包含复杂的、列表类型的数据结构，如聊天历史 (messages)。如果处理不当，状态更新可能会互相覆盖，导致信息丢失。LangGraph 的 Reducer 机制解决了这个问题，确保了状态的安全聚合。

## 4.1 列表状态问题与 Reducer 解决方案
当状态中的某个键是一个列表时，如果一个节点返回了整个列表的更新，它会覆盖掉前一个节点对该列表所做的修改。尤其在并行工作流中，多个节点可能同时尝试向列表追加内容，如果仅依靠简单的覆盖逻辑，会导致状态的不确定性。

**Reducer** 机制应运而生。它是一种绑定到状态字段上的函数，定义了 LangGraph 如何将节点返回的**增量更新** 合并到该字段的现有值中。Reducer 确保了即使在并行执行中，状态也能按预定义的逻辑安全地、一致地聚合。

## 4.2 使用 Annotated 和内置 add_messages Reducer
对于处理聊天消息历史，LangGraph 提供了内置的 add_messages Reducer。这个 Reducer 专门用于将新的 BaseMessage 对象安全地追加到状态中的消息列表中，避免了覆盖问题。

在 TypedDict 状态定义中，我们通过 Python 的 Annotated 类型提示语法，将 Reducer 函数声明式地绑定到特定的键上。这种设计模式将数据模式 (TypedDict) 和数据行为 (Reducer) 结合在一起，允许所有未来的节点专注于计算本身，而无需担心如何安全地更新共享列表。

## 实战示例：状态中消息列表的自动追加管理 (Message List Aggregation)
此示例演示了如何使用 add_messages Reducer 来管理聊天历史，确保多步响应都能正确地追加到 messages 列表中。

Mermaid 图表 - 状态聚合流程：

该图显示了一个线性流程，但其重点在于状态内部的消息列表是如何通过 Reducer 机制安全累加的。

```python
# 确保安装：pip install -U langgraph langchain-core
from typing import TypedDict, List
from typing_extensions import Annotated
from langgraph.graph import StateGraph, START, END
from langgraph.graph.message import add_messages
from langchain_core.messages import BaseMessage, HumanMessage, AIMessage

# 1. 定义状态：使用 Annotated 绑定 Reducer
class ChatState(TypedDict):
    # messages 键将使用 add_messages Reducer 自动追加内容
    messages: Annotated[List[BaseMessage], add_messages]
    step_count: int

# 2. 节点 A: 初始响应
def node_agent_1(state: ChatState) -> dict:
    new_message = AIMessage("Agent 1: 已经处理了你的请求，正在交给 Agent 2 确认。")
    print("Agent 1: Adding response to state.")
    # 返回更新，Reducer 会自动处理列表追加
    return {"messages": [new_message], "step_count": state["step_count"] + 1}

# 3. 节点 B: 第二次响应
def node_agent_2(state: ChatState) -> dict:
    new_message = AIMessage("Agent 2: 确认无误，流程结束。")
    print("Agent 2: Adding confirmation to state.")
    return {"messages": [new_message], "step_count": state["step_count"] + 1}

# 4. 构建图
workflow = StateGraph(ChatState)
workflow.add_node("agent_1", node_agent_1)
workflow.add_node("agent_2", node_agent_2)

workflow.set_entry_point("agent_1")
workflow.add_edge("agent_1", "agent_2")
workflow.add_edge("agent_2", END)

app = workflow.compile()

# 运行流程
initial_input: ChatState = {
    "messages": [HumanMessage(content="请帮我开始一个两步流程。")],
    "step_count": 0
}
print("--- 开始运行带 Reducer 的流程 ---")
final_output = app.invoke(initial_input)

# 检查最终消息列表：应包含 1 (Human), 2 (Agent 1), 3 (Agent 2)
print("\nFinal State Messages:")
for i, msg in enumerate(final_output["messages"]):
    print(f"[{i+1}] {msg.type.upper()}: {msg.content}")

print(f"\nTotal steps: {final_output['step_count']}") # 预期为 2
```
