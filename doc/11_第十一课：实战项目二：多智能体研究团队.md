## 目标
构建一个由"主管" (Supervisor) 协调的、多智能体协作的图。这是 LangGraph 最擅长的领域之一。

**核心讲解：**
我们将构建一个"研究团队"，它包含：
- **Researcher (研究员)**：负责使用搜索工具查找资料。
- **Writer (撰写员)**：负责将研究资料整合成一份报告。
- **Supervisor (主管)**：整个团队的"路由器"。它接收用户的初始请求，决定先把任务交给谁（例如 Researcher）。当一个智能体完成工作后，它会把结果交还给 Supervisor，Supervisor 再决定下一步交给谁（例如交给 Writer，或者如果工作完成了，就 END）。

## 关键技术点
我们会为 Researcher 和 Writer 分别创建独立的 StateGraph (就像课程六的 Agent)。

然后我们会使用 `workflow.add_node("name", agent.compile())` 将这些"子图" (Sub-Graphs) 作为节点添加到"父图" (Supervisor 图) 中。

Supervisor 图的核心将是一个强大的 `add_conditional_edges`，它充当了团队的"路由器"。

## 实战代码
```python
import operator
from typing import Annotated, TypedDict, List, Literal
from langgraph.graph import StateGraph, END
from langchain_openai import ChatOpenAI
from langchain_core.messages import BaseMessage, HumanMessage, AIMessage, ToolMessage
from langchain_core.tools import tool
from langchain_community.tools import DuckDuckGoSearchRun
from langgraph.prebuilt import ToolExecutor

# --- 1. 定义工具 (供 Researcher 使用) ---
@tool
def search(query: str):
    """搜索互联网获取信息。"""
    print(f"--- [工具] Researcher 正在搜索: {query} ---")
    return DuckDuckGoSearchRun().run(query)

# --- 2. 定义 "Agent" 节点 (可复用的) ---
# 这是一个标准的 Agent 节点创建函数，用于创建 'Researcher' 和 'Writer'
def create_agent_node(llm, tools, system_prompt):
    # 将工具绑定到 LLM
    llm_with_tools = llm.bind_tools(tools)
    
    def agent_node(state: AgentState):
        print(f"--- [节点] 正在调用 {system_prompt[:20]}... ---")
        
        # 为了让 Agent 知道自己的角色，我们在消息列表的 *开头* 插入系统提示
        # (注：更优化的方式是在 State 中增加 'system_prompt' 字段)
        messages = [HumanMessage(content=system_prompt)] + state['messages']
        
        response = llm_with_tools.invoke(messages)
        # 我们只返回 AIMessage 或 ToolMessage，不返回系统提示
        return {"messages": [response]}
        
    return agent_node

# --- 3. 定义 "Tool" 节点 (可复用的) ---
# (与项目一相同)
def create_tool_node(tools):
    tool_executor = ToolExecutor(tools)
    
    def tool_node(state: AgentState):
        print("--- [节点] Tool Executor ---")
        last_message = state['messages'][-1]
        tool_messages = tool_executor.invoke(last_message)
        return {"messages": tool_messages}
        
    return tool_node

# --- 4. 定义 "Agent" 状态和决策函数 ---
class AgentState(TypedDict):
    """
    智能体状态定义
    用于研究员和撰写员子图
    """
    messages: Annotated[List[BaseMessage], operator.add]

def should_continue(state: AgentState):
    """
    决定是否继续工具调用
    
    Args:
        state: 当前状态
        
    Returns:
        Literal: "continue" 表示继续，"end" 表示结束
    """
    
    if not state['messages'][-1].tool_calls:
        return "end"
    else:
        return "continue"

# --- 5. 创建 "Researcher" 子图 ---
researcher_llm = ChatOpenAI(temperature=0)
researcher_tools = [search]
researcher_prompt = """
你是一名专业的研究员。你的任务是使用搜索工具查找信息，并简洁地总结你的发现。

工作流程：
1. 分析用户的问题，确定需要搜索的关键信息
2. 使用search工具进行搜索
3. 整理搜索结果，提供清晰简洁的总结
4. 明确指出你的研究发现，不要编造答案

输出格式：
- 先提供搜索结果总结
- 然后给出关键发现
- 最后说明研究状态
"""

# 创建节点
researcher_agent_node = create_agent_node(researcher_llm, researcher_tools, researcher_prompt)
researcher_tool_node = create_tool_node(researcher_tools)

# 创建图
researcher_workflow = StateGraph(AgentState)
researcher_workflow.add_node("agent", researcher_agent_node)
researcher_workflow.add_node("tools", researcher_tool_node)
researcher_workflow.set_entry_point("agent")
researcher_workflow.add_conditional_edges(
    "agent",
    should_continue,
    {"continue": "tools", "end": END}
)
researcher_workflow.add_edge("tools", "agent")

# 编译子图
researcher_graph = researcher_workflow.compile()
print("--- 'Researcher' 子图已编译 ---")

# --- 6. 创建 "Writer" 子图 ---
writer_llm = ChatOpenAI(temperature=0)
writer_prompt = """
你是一名专业的科技报告撰写者。你的任务是根据提供的信息（通常来自研究员），撰写一份结构清晰、易于理解的报告。

撰写要求：
1. 分析提供的信息内容
2. 撰写结构清晰的报告，包括：
   - 标题
   - 摘要
   - 主要内容
   - 结论
3. 使用简洁专业的语言
4. 不要使用工具，只根据已有的对话内容撰写
5. 明确说明报告撰写完成

报告格式示例：
# 主题报告

## 摘要
[简要概述研究发现]

## 主要内容
[详细描述研究结果]

## 结论
[总结主要发现和洞察]

## 报告撰写状态
✅ 报告已完成
"""

# Writer 没有工具
writer_agent_node = create_agent_node(writer_llm, [], writer_prompt)

# 创建图 (只有一个节点)
writer_workflow = StateGraph(AgentState)
writer_workflow.add_node("agent", writer_agent_node)
writer_workflow.set_entry_point("agent")
writer_workflow.add_edge("agent", END) # Writer 思考一次就结束

# 编译子图
writer_graph = writer_workflow.compile()
print("--- 'Writer' 子图已编译 ---")

# --- 7. 定义 "Supervisor" (主管) 图 ---

# 主管图的状态：
# 'messages' 是共享的对话历史
# 'next_agent' 是主管决定下一个由谁来工作的指示
class SupervisorState(TypedDict):
    """
    主管状态定义
    用于主协调图
    """
    messages: Annotated[List[BaseMessage], operator.add]
    next_agent: Literal["Researcher", "Writer", "END"]

# Supervisor 的 "路由器" (决策函数)
def supervisor_router(state: SupervisorState):
    print("--- [节点] Supervisor (Router) ---")
    
    # 获取最后一条消息
    last_message = state['messages'][-1]
    
    # 如果最后一条消息是 AIMessage (来自某个 Agent)
    if isinstance(last_message, AIMessage):
        content = last_message.content
        # 检查是否包含总结或研究相关词汇
        research_keywords = ["总结", "发现", "研究结果", "通过搜索", "找到了", "研究"]
        completion_keywords = ["完成了", "结束", "完成", "结束任务"]
        
        if any(keyword in content for keyword in research_keywords):
            print("[主管路由] -> Writer (研究完成)")
            return "Writer"
        elif any(keyword in content for keyword in completion_keywords):
            print("[主管路由] -> END (任务完成)")
            return "END"
        else:
            print("[主管路由] -> Researcher (需要更多研究)")
            return "Researcher"
    
    # 如果最后一条消息是 HumanMessage (初始请求)
    else:
        print("--- [主管决策] -> Researcher ---")
        return "Researcher"

# --- 8. 构建并编译 "Supervisor" (主管) 图 ---
super_workflow = StateGraph(SupervisorState)

supervisor_llm = ChatOpenAI(temperature=0)
supervisor_prompt = """你是一个多智能体团队的主管。你的工作是根据当前的对话状态，决定下一个应该由哪个团队成员来执行任务。
可选的团队成员有： 'Researcher', 'Writer'。
- 如果任务是收集信息或回答事实性问题，请选择 'Researcher'。
- 如果任务是基于已有信息撰写报告或总结，请选择 'Writer'。
- 如果任务已完成，请选择 'END'。

请只返回下一个团队成员的名字： 'Researcher', 'Writer', 或 'END'。"""

# 主管节点：调用 LLM 来决定下一步
def supervisor_node(state: SupervisorState):
    print("--- [节点] Supervisor ---")
    # 构建给主管的提示
    prompt_messages = [
        HumanMessage(content=supervisor_prompt),
        # 添加最后几条消息作为上下文
        *state['messages'][-3:] 
    ]
    response = supervisor_llm.invoke(prompt_messages)
    
    next_agent = response.content.strip()
    print(f"--- [主管决策] -> {next_agent} ---")
    
    # 如果 LLM 没返回正确的名字，就结束
    if next_agent not in ["Researcher", "Writer"]:
        next_agent = "END"
        
    return {"next_agent": next_agent}

# 添加主管节点
super_workflow.add_node("Supervisor", supervisor_node)

# !! 重点: 将子图作为节点添加到主管图中 !!
super_workflow.add_node("Researcher", researcher_graph)
super_workflow.add_node("Writer", writer_graph)

# 设置入口点
super_workflow.set_entry_point("Supervisor")

# 添加条件边 (主管的"路由")
super_workflow.add_conditional_edges(
    "Supervisor",
    supervisor_router,
    # "END" 路由到图的终点
    {"Researcher": "Researcher", "Writer": "Writer", "END": END}
)

# 两个子图执行完毕后，*总是* 返回给主管做下一步决策
super_workflow.add_edge("Researcher", "Supervisor")
super_workflow.add_edge("Writer", "Supervisor")

# --- 9. 编译并运行主管图 ---
app = super_workflow.compile()
print("--- 主管 (Supervisor) 图已编译 ---")

# --- 运行 ---
print("\n--- 开始运行多智能体团队 ---")
query = "LangGraph 和 LangChain 的区别是什么？请给我一份简短的报告。"
inputs = {"messages": [HumanMessage(content=query)]}

# 使用 stream() 来查看每一步的状态
# (invoke 也可以，但 stream 能更好地展示流程)
final_state = {}
for event in app.stream(inputs, config={"recursion_limit": 10}):
    for key, value in event.items():
        print(f"--- [Stream Event] 节点: {key} ---")
        if 'messages' in value:
            # 打印最新的一条消息
            print(f"最新消息: {value['messages'][-1].content}\n")
            final_state = value # 捕获最后一个状态
        
print("\n--- 多智能体团队运行结束 ---")
print("--- 最终报告 (来自 Writer): ---")
# 最终的报告通常是 Writer 发出的最后一条 AIMessage
print(final_state['messages'][-2].content) # (-1 可能是 Supervisor 的 'END' 决定)
```

## 重点总结
- **分层图 (Hierarchical Graphs)**：我们创建了两个“子图” (researcher_graph, writer_graph)，并将它们作为节点添加到“父图” (super_workflow) 中。
- **add_node("Name", compiled_subgraph)**：这是实现分层图的关键语法。
- **Supervisor as Router**：主管 (Supervisor) 的核心作用是一个“路由器”。它不自己干活，它只决定下一个谁来干活。
- **循环**：Agent -> Supervisor -> Agent -> Supervisor ... 这种模式是多智能体协作的核心。每个智能体完成工作后都向主管“汇报”，主管再进行“派单”。
- **状态共享**：所有的图（父图和子图）共享同一个 State 结构 (SupervisorState 兼容 AgentState)，这就是它们能够协作的基础——它们都在读取和写入同一个“白板” (messages)。
