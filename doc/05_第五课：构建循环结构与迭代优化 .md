循环结构是 LangGraph 实现自校正和迭代优化 Agent 的核心机制。通过将条件边缘指向自身或其他前置节点，可以创建强大的反馈回路。

## 5.1 核心模式：生成器-评论家 (Generator-Critic) 架构

Generator-Critic 模式是实现 AI 迭代优化的标准模式。工作流中包含：

**Generator (生成器)：** 负责根据输入和历史反馈生成答案或内容。

**Critic (评论家)：** 负责评估生成器的输出。

**Conditional Edge (条件边缘)：** 根据评论家的评估结果，决定是将流程路由回 Generator 进行修改，还是路由到 END 结束流程。

这种模式广泛应用于代码生成、写作精炼和复杂问题求解等场景，提高了最终输出的可靠性。

## 5.2 实现循环路由：将条件边缘指向自身或其他前置节点

实现循环的关键在于 add_conditional_edges 的 path_map 参数中，包含一个指向源节点或前置节点的路径。例如，从 critic 节点出发，如果评估结果是 "NEEDS_REVISION"，流程应返回到 "generator" 节点。

## 5.3 循环终止条件：通过状态控制 END 路由

成功的 Agent 循环必须有明确的终止机制，以防止无限循环。终止条件通常通过检查状态中的两个关键信息来实现：

**LLM 的语义判断：** 评论家节点是否在状态中标记了"满意"或"已通过"的反馈。

**硬性限制：** 是否达到了预设的最大迭代次数 (revision_number)。

循环的有效性完全依赖于状态的持久性和丰富性。只有当状态安全地记录了历史和错误（例如，通过模块四讨论的 Reducer 累加 critique_history），Generator 才能实现有意义的**反思 (Reflection)** 和改进。

## 实战示例：带有固定迭代次数限制的答案精炼循环

此示例模拟一个 Agent 尝试回答问题，最多迭代 3 次，直到达到最大尝试次数或评论家满意为止。

**自校正循环：LangGraph流程图**

```python
from typing import TypedDict, List
from langgraph.graph import StateGraph, END

# 1. 定义循环状态
class RevisionState(TypedDict):
    prompt: str
    current_answer: str
    critique_history: List[str]
    revision_number: int # 追踪迭代次数

# 2. 节点 A: 生成器 (模拟实际的LLM调用)
def generator_node(state: RevisionState) -> dict:
    rev_num = state["revision_number"] + 1
    # 基于历史和当前答案生成新答案 
    new_answer = f"Attempt {rev_num}: Draft for '{state['prompt']}'"
    if state["critique_history"]:
        new_answer += f" (Revised based on critique: {state['critique_history'][-1]})"
    
    # 模拟生成新内容
    print(f"--- GENERATING (Rev {rev_num}) ---")
    return {"current_answer": new_answer, "revision_number": rev_num}

# 3. 节点 B: 评论家 (模拟LLM评估)
MAX_REVISIONS = 3
def critic_node(state: RevisionState) -> dict:
    # 模拟评论逻辑：如果未达到最大次数，则需要修改
    if state["revision_number"] < MAX_REVISIONS:
        critique = "NEEDS_REVISION: The answer lacks detail. Try again."
    else:
        critique = "SATISFIED: Maximum revisions reached or quality deemed acceptable."

    next_step = {"critique_history": state["critique_history"] + [critique]}
    
    print(f"--- CRITIQUE: {critique.split(':')} ---")
    return next_step

# 4. 路由函数：决定是否循环
def route_after_critique(state: RevisionState) -> str:
    # 检查最新的评论
    if "NEEDS_REVISION" in state["critique_history"][-1]:
        return "generator" # 路由回 Generator
    else:
        return END # 结束流程

# 5. 构建图并定义循环边缘
workflow = StateGraph(RevisionState)
workflow.add_node("generator", generator_node)
workflow.add_node("critic", critic_node)

workflow.set_entry_point("generator")
workflow.add_edge("generator", "critic") # 固定从 Generator 到 Critic

# 定义条件循环边缘
workflow.add_conditional_edges(
    "critic",
    route_after_critique,
    {
        "generator": "generator", # 满足 NEED_REVISION 时，返回 generator
        END: END                 # 满足 SATISFIED 时，结束
    }
)

app = workflow.compile()

# 运行流程
initial_input: RevisionState = {
    "prompt": "Explain LangGraph's core loop concept.",
    "current_answer": "", 
    "critique_history": [], 
    "revision_number": 0
}

print("--- 开始运行自校正循环 (Max 3次迭代) ---")
final_output = app.invoke(initial_input)

print("\nFinal State Output:")
print(f"Final Answer: {final_output['current_answer']}")
print(f"Total Revisions: {final_output['revision_number']}") # 预期为 3
print(f"Critique History: {final_output['critique_history']}")
```

## 进一步扩展

### 1. 添加真实LLM集成

替换模拟的生成器和评论家函数：

```python
import openai  # 或其他LLM库

def generator_node(state: RevisionState) -> Dict[str, Any]:
    # 使用真实LLM生成内容
    prompt = f"Generate a detailed answer for: {state['prompt']}"
    if state['critique_history']:
        prompt += f"\nPrevious feedback: {state['critique_history'][-1]}"
    
    response = openai.ChatCompletion.create(
        model="gpt-3.5-turbo",
        messages=[{"role": "user", "content": prompt}]
    )
    
    new_answer = response.choices[0].message.content
    return {
        "current_answer": new_answer,
        "revision_number": state["revision_number"] + 1
    }
```

### 2. 添加更复杂的评价标准

```python
def critic_node(state: RevisionState, max_revisions: int = 3) -> Dict[str, Any]:
    # 使用多个评价维度
    criteria = {
        "completeness": len(state["current_answer"]) > 100,
        "clarity": "?" in state["current_answer"] or "." in state["current_answer"],
        "relevance": state["prompt"].lower() in state["current_answer"].lower()
    }
    
    all_good = all(criteria.values())
    
    if not all_good and state["revision_number"] < max_revisions:
        missing = [k for k, v in criteria.items() if not v]
        critique = f"NEEDS_REVISION: Missing or inadequate {', '.join(missing)}"
    else:
        critique = "SATISFIED: All quality criteria met."
    
    return {"critique_history": state["critique_history"] + [critique]}
```

### 3. 添加用户交互功能

```python
def human_critic_node(state: RevisionState) -> Dict[str, Any]:
    print(f"\n当前答案: {state['current_answer']}")
    满意 = input("这个答案满意吗? (y/n): ").strip().lower()
    
    if 满意 in ['y', 'yes', '是']:
        critique = "SATISFIED: User approved the answer."
    else:
        feedback = input("请提供改进建议: ").strip()
        critique = f"NEEDS_REVISION: {feedback}"
    
    return {"critique_history": state["critique_history"] + [critique]}
```
