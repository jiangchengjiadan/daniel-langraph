动态控制流是 LangGraph 实现 Agent 智能的关键所在。通过条件边缘，流程允许图根据当前状态动态地选择下一个要执行的节点。

## 3.1 条件边缘机制：add_conditional_edges 的原理

条件边缘的核心在于引入了一个路由函数，该函数在源节点执行完毕后运行，根据当前状态决定下一步的走向。实现条件边缘使用 `add_conditional_edges` 方法，它需要指定三个关键参数：源节点名称、决策函数，以及一个可选的路径映射字典。

它包含三个部分：

**起始节点 (Start Node):** 决策从哪个节点之后开始。

**决策函数 (Decision Function):** 一个函数，它接收当前 state，然后返回一个**字符串**，这个字符串就是下一个节点的名字。

**输出映射 (path map):** 一个字典，将决策函数可能返回的名字映射到图中的实际节点。

## 3.2 决策函数：根据状态内容进行决策

决策函数接受当前的完整 State 作为输入。它的输出必须是一个可哈希值（通常是字符串），代表下一个目标节点的名称，或者显式返回 `END` 来终止流程。

一个重要的架构原则是，决策函数应该严格遵循**单一职责原则**：它只负责决策流程路径，**不应修改状态**。所有状态的修改和副作用都应保留给节点 (Node) 完成。

这种分离确保了流程路径和数据副作用是解耦的，使得调试更加简单，并避免了在并发环境中可能出现的状态冲突。

## 3.3 输出映射 (path_map)：将路由结果映射到目标节点

`path_map` 是一个可选的字典，它将决策函数返回的抽象值（例如，"needs_search" 或 "direct_answer"）映射到图中的实际节点名称。如果路由函数直接返回节点名称，可以省略 `path_map`。

使用 `path_map` 的优势在于，它使得决策函数的代码逻辑更加清洁，将决策逻辑与图的结构定义分离。

此外，动态控制流的强大之处在于它不仅支持分支（选择一个路径），还支持决策函数返回一个目标节点列表，从而在下一个"超级步骤"中并行执行多个任务。这是构建多 Agent 系统（例如，同时启动一个研究和一个计算 Agent）的基础。

## 实战示例 1：基于输入条件的简单决策路由器 (Decision Router)

此示例根据输入的数字大小，将流程动态分发到"快速通道"或"复杂通道"。

```python
from typing import TypedDict, Literal
from langgraph.graph import StateGraph, START, END

# 1. 定义状态
class RoutingState(TypedDict):
    input_number: int
    path_taken: Literal["fast", "complex", "error"]

# 2. 节点 A: 初始处理
def initial_process(state: RoutingState) -> dict:
    # 模拟一些预处理逻辑，不修改关键数据
    print("Node A: Initial validation complete.")
    return {}

# 3. 决策函数：检查数字大小 (不修改状态)
def decide_route(state: RoutingState) -> str:
    # 路由函数只读取状态，并返回决策结果（字符串）
    if state["input_number"] > 100:
        print(f"Routing: Number {state['input_number']} > 100. Taking 'complex' path.")
        return "COMPLEX" 
    else:
        print(f"Routing: Number {state['input_number']} <= 100. Taking 'fast' path.")
        return "FAST"

# 4. 节点 B: 快速通道 (更新状态)
def fast_route_node(state: RoutingState) -> dict:
    print("Node B: Executing Fast Route. Result is simple.")
    return {"path_taken": "fast"}

# 5. 节点 C: 复杂通道 (更新状态)
def complex_route_node(state: RoutingState) -> dict:
    print("Node C: Executing Complex Route. Result requires heavy computation.")
    return {"path_taken": "complex"}

# 6. 构建图并定义边缘
workflow = StateGraph(RoutingState)
workflow.add_node("initial_process", initial_process)
workflow.add_node("fast_route", fast_route_node)
workflow.add_node("complex_route", complex_route_node)

workflow.set_entry_point("initial_process")

# 添加条件边缘：根据 decide_route 的返回值跳转
workflow.add_conditional_edges(
    "initial_process", # 源节点
    decide_route,      # 路由函数
    {
        "FAST": "fast_route",     # 路由函数返回 "FAST" 时，跳转到 fast_route 节点
        "COMPLEX": "complex_route" # 路由函数返回 "COMPLEX" 时，跳转到 complex_route 节点
    }
)

# 终结流程：无论走哪条路，都连接到 END
workflow.add_edge("fast_route", END)
workflow.add_edge("complex_route", END)

app = workflow.compile()

# 运行 1 (Fast Route)
print("--- Run 1: Input 50 (Fast) ---")
final_output_1 = app.invoke({"input_number": 50, "path_taken": "error"})
print("Result 1 Path:", final_output_1["path_taken"])

# 运行 2 (Complex Route)
print("\n--- Run 2: Input 200 (Complex) ---")
final_output_2 = app.invoke({"input_number": 200, "path_taken": "error"})
print("Result 2 Path:", final_output_2["path_taken"])
```

## 实战代码 2

我们将构建一个"智能分类路由"：

- **节点A (classify_input):** 判断用户输入是"问候"还是"问题"
- **条件边:** 根据分类结果，路由到...
- **节点B (greet_node):** 如果是问候，就回复问候
- **节点C (answer_node):** 如果是问题，就回复"我正在查找答案"

```python
from typing import TypedDict
from langgraph.graph import StateGraph, END

# --- 1. 定义状态 (State) ---
class RouterState(TypedDict):
    user_input: str      # 原始输入
    classification: str  # 分类结果 ("greeting" or "question")
    response: str        # 最终的回复

# --- 2. 定义节点 (Nodes) ---
def classify_input(state: RouterState):
    print("--- 正在执行 classify_input ---")
    user_input = state['user_input'].lower()
    
    if "你好" in user_input or "嗨" in user_input:
        classification = "greeting"
    else:
        classification = "question"
        
    print(f"分类结果: {classification}")
    return {"classification": classification}

def greet_node(state: RouterState):
    print("--- 正在执行 greet_node ---")
    return {"response": "你好！很高兴见到你。"}

def answer_node(state: RouterState):
    print("--- 正在执行 answer_node ---")
    return {"response": "你提了一个好问题。我需要思考一下..."}

# --- 3. 定义决策函数 (Decision Function) ---
# 这个函数 *读取* 状态，然后 *返回* 一个字符串
# 这个字符串将用于决定下一步去哪里
def should_i_respond(state: RouterState):
    print("--- 正在做决策 ---")
    classification = state['classification']
    
    if classification == "greeting":
        # 这个返回的 "greet" 字符串必须在下面的 map 中有对应
        return "greet"
    else:
        return "answer"

# --- 4. 定义图 (Graph) ---
workflow = StateGraph(RouterState)

# 添加节点
workflow.add_node("classify", classify_input)
workflow.add_node("greet", greet_node)
workflow.add_node("answer", answer_node)

# --- 5. 定义图的结构 (Edges) ---
workflow.set_entry_point("classify")

# 添加条件边！
workflow.add_conditional_edges(
    # 起始节点: 'classify' 节点执行完毕后，开始做决策
    "classify",
    # 决策函数: 'should_i_respond'
    should_i_respond,
    # 目标映射:
    {
        # 如果决策函数返回 "greet"，则下一步去 'greet' 节点
        "greet": "greet",
        # 如果决策函数返回 "answer"，则下一步去 'answer' 节点
        "answer": "answer"
    }
)

# 两个分支最终都汇入 END
workflow.add_edge("greet", END)
workflow.add_edge("answer", END)

# --- 6. 编译图 ---
app = workflow.compile()

# --- 7. 运行图 (测试两次) ---

# 测试1: 问候
print("--- 测试 1: 问候 ---")
inputs_1 = {"user_input": "你好啊，机器人！"}
final_state_1 = app.invoke(inputs_1)
print(f"最终回复: {final_state_1['response']}")

print("\n" + "="*20 + "\n")

# 测试2: 问题
print("--- 测试 2: 问题 ---")
inputs_2 = {"user_input": "LangGraph 是如何工作的？"}
final_state_2 = app.invoke(inputs_2)
print(f"最终回复: {final_state_2['response']}")

# --- 预期输出 ---
# --- 测试 1: 问候 ---
# --- 正在执行 classify_input ---
# 分类结果: greeting
# --- 正在做决策 ---
# --- 正在执行 greet_node ---
# 最终回复: 你好！很高兴见到你。
#
# ====================
#
# --- 测试 2: 问题 ---
# --- 正在执行 classify_input ---
# 分类结果: question
# --- 正在做决策 ---
# --- 正在执行 answer_node ---
# 最终回复: 你提了一个好问题。我需要思考一下...
```

## 本节重点

- **决策函数不更新状态:** `should_i_respond` 只是读取 state，它返回的是一个**字符串**（下一个节点的路由键）。
- **`add_conditional_edges` 是核心。** 它需要：起始节点、决策函数、和一个 `{ "路由键": "目标节点名" }` 的字典。

这是构建智能体的基础：根据 LLM 的输出（例如，是否需要调用工具）来决定下一步的走向。
