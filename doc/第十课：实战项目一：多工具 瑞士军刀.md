## 目标

综合运用**条件边** 和**工具调用** ，构建一个能根据需要，在_多个_ 工具中动态选择一个或多个来执行的智能体。

这在课程六的基础上做了一个重要升级：

课程六：Agent -> (是否用工具?) -> Tool

本项目：Agent -> (用哪个工具? A, B, 还是 C?) -> Tool Router -> (Tool A, B, C)

我们将使用 LangGraph 内置的 ToolExecutor和 ToolNode两种方式来简化工具的执行和路由。

---

## ToolExecutor方式

Agent (LLM) 负责生成 tool_calls，ToolExecutor 负责根据 tool_calls 中的 name 字段来调用正确的函数。

我们的决策逻辑 should_continue 保持不变，但 tool_node 会变得更健壮。

### 实战代码

我们将给 Agent 两把"瑞士军刀"：

**search (DuckDuckGo):** 用于搜索时事、信息

**get_word_length:** 一个自定义的、简单的 Python "工具"，用于计算单词长度

```python
import operator
from typing import Annotated, TypedDict, List
from langgraph.graph import StateGraph, END
from langgraph.checkpoint.sqlite import SqliteSaver # (可选, 用于持久化)
from langgraph.prebuilt import ToolExecutor # !! 导入预构建的 ToolExecutor
from langchain_openai import ChatOpenAI
from langchain_core.messages import BaseMessage, HumanMessage, AIMessage, ToolMessage
from langchain_core.tools import tool
from langchain_community.tools import DuckDuckGoSearchRun

# --- 1. 定义工具 ---

# 工具1: 搜索
search_tool = DuckDuckGoSearchRun()

# 工具2: 计算单词长度
@tool
def get_word_length(word: str):
    """计算一个单词中字母的数量。"""
    print(f"--- [工具] 正在计算 '{word}' 的长度 ---")
    return len(word)

# 工具列表
tools = [search_tool, get_word_length]

# --- 2. 实例化 ToolExecutor ---
# ToolExecutor 会帮我们处理工具的调用和结果封装
tool_executor = ToolExecutor(tools)

# --- 3. 绑定工具到 LLM ---
llm = ChatOpenAI(temperature=0)
llm_with_tools = llm.bind_tools(tools)

# --- 4. 定义状态 (State) ---
class AgentState(TypedDict):
    messages: Annotated[List[BaseMessage], operator.add]

# --- 5. 定义节点 (Nodes) ---

# 节点1: Agent (LLM)
def agent_node(state: AgentState):
    print("--- [节点] Agent ---")
    response = llm_with_tools.invoke(state['messages'])
    return {"messages": [response]}

# 节点2: Tool Executor 节点
# 这个节点现在使用 ToolExecutor 来简化操作
def tool_node(state: AgentState):
    print("--- [节点] Tool Executor ---")
    last_message = state['messages'][-1]
    
    # 使用 ToolExecutor.invoke()
    # 它会根据 last_message 中的 tool_calls 自动调用正确的工具
    tool_messages = tool_executor.invoke(last_message)
    
    # tool_messages 是一个 ToolMessage 列表
    return {"messages": tool_messages}

# --- 6. 定义决策函数 (Conditional Edge) ---
def should_continue(state: AgentState):
    print("--- [决策] Should Continue? ---")
    if not state['messages'][-1].tool_calls:
        print("--- [决策] -> END ---")
        return "end"
    else:
        print("--- [决策] -> Tool ---")
        return "continue"

# --- 7. 定义并编译图 ---
workflow = StateGraph(AgentState)
workflow.add_node("agent", agent_node)
workflow.add_node("tools", tool_node)
workflow.set_entry_point("agent")
workflow.add_conditional_edges(
    "agent",
    should_continue,
    {"continue": "tools", "end": END}
)
workflow.add_edge("tools", "agent")

app = workflow.compile()
# (可选) 添加持久化
# checkpointer = SqliteSaver.memory()
# app = workflow.compile(checkpointer=checkpointer)

# --- 8. 运行图 ---
print("--- 测试 1: 使用 'get_word_length' 工具 ---")
inputs_1 = {"messages": [HumanMessage(content="单词 'LangGraph' 有多长?")]}
# config = {"configurable": {"thread_id": "thread-1"}}
response_1 = app.invoke(inputs_1) #, config=config)
print(f"AI: {response_1['messages'][-1].content}\n")

print("--- 测试 2: 使用 'search' 工具 ---")
inputs_2 = {"messages": [HumanMessage(content="LangGraph 最新的更新是什么?")]}
# config = {"configurable": {"thread_id": "thread-2"}}
response_2 = app.invoke(inputs_2) #, config=config)
print(f"AI: {response_2['messages'][-1].content}\n")

print("--- 测试 3: 不使用工具 ---")
inputs_3 = {"messages": [HumanMessage(content="你好")]}
# config = {"configurable": {"thread_id": "thread-3"}}
response_3 = app.invoke(inputs_3) #, config=config)
print(f"AI: {response_3['messages'][-1].content}\n")

# --- 预期输出 ---
# --- 测试 1: 使用 'get_word_length' 工具 ---
# --- [节点] Agent ---
# --- [决策] Should Continue? ---
# --- [决策] -> Tool ---
# --- [节点] Tool Executor ---
# --- [工具] 正在计算 'LangGraph' 的长度 ---
# --- [节点] Agent ---
# --- [决策] Should Continue? ---
# --- [决策] -> END ---
# AI: 单词 'LangGraph' 的长度是 9。
#
# --- 测试 2: 使用 'search' 工具 ---

# --- [决策] Should Continue? ---
# --- [决策] -> Tool ---
# --- [节点] Tool Executor ---
# (DuckDuckGoSearchRun 的输出)
# --- [节点] Agent ---
# --- [决策] Should Continue? ---
# --- [决策] -> END ---

#
# --- 测试 3: 不使用工具 ---
# --- [节点] Agent ---
# --- [决策] Should Continue? ---
# --- [决策] -> END ---
```

### ToolExecutor使用重点

**ToolExecutor(tools):** 它是管理和执行多个工具的"瑞士军刀"管家。你只需把工具列表扔给它。

**tool_node 的简化** : 对比课程四，我们的 tool_node 变得极其简单，它只管调用 tool_executor.invoke()。

**LLM 的动态决策** : 强大的 llm_with_tools 会自动分析用户的提问，并在其"工具箱" (tools 列表) 中选择_最合适_ 的工具来调用。

---

## ToolNode方式

ToolExecutor方式: 开发者需要手动解析 AIMessage 的 tool_calls 字段，调用工具，然后手动构造 ToolMessage 返回给状态。

ToolNode方式:

1. 自动解析: 自动识别上一个消息中的 tool_calls。
2. 并行执行: 如果 Agent 同时请求调用三个工具，ToolNode 会并发处理。
3. 状态无缝同步: 执行结果会自动作为 ToolMessage 列表合并到 State 的 messages 中。

### 实战代码

还是上面的例子，采用ToolNode方式。

```python
import operator
from typing import Annotated, Sequence, TypedDict, Union

from langchain_openai import ChatOpenAI
from langchain_core.messages import BaseMessage, HumanMessage
from langchain_core.tools import tool
from langchain_community.tools import DuckDuckGoSearchRun
from langgraph.graph import StateGraph, END
from langgraph.prebuilt import ToolNode  # !! 官方推荐的工具节点封装

# --- 1. 定义工具 (Tools) ---
@tool
def get_word_length(word: str):
    """计算一个单词中字母的数量。"""
    print(f"--- [工具执行] 计算 '{word}' 的长度 ---")
    return len(word)

search_tool = DuckDuckGoSearchRun()
tools = [search_tool, get_word_length]

# --- 2. 状态定义 ---
class AgentState(TypedDict):
    # 使用 operator.add 允许消息列表不断累加
    messages: Annotated[Sequence[BaseMessage], operator.add]

# --- 3. 初始化模型与工具节点 ---
llm = ChatOpenAI(model="gpt-4o", temperature=0)
# 将工具定义绑定到模型，使其能够生成 tool_calls
llm_with_tools = llm.bind_tools(tools)

# 创建标准的工具节点
tool_node = ToolNode(tools)

# --- 4. 节点逻辑 (Nodes) ---
def agent_node(state: AgentState):
    """Agent 决定下一步动作"""
    print("--- [节点] Agent 决策中 ---")
    response = llm_with_tools.invoke(state['messages'])
    return {"messages": [response]}

# --- 5. 决策函数 (Router) ---
def should_continue(state: AgentState):
    """根据模型是否生成 tool_calls 判断去向"""
    last_message = state['messages'][-1]
    if not last_message.tool_calls:
        return "end"
    return "continue"

# --- 6. 构建图 (Graph) ---
workflow = StateGraph(AgentState)

# 添加节点
workflow.add_node("agent", agent_node)
workflow.add_node("tools", tool_node) # 直接添加 ToolNode

# 设置入口
workflow.set_entry_point("agent")

# 设置条件边
workflow.add_conditional_edges(
    "agent",
    should_continue,
    {
        "continue": "tools", 
        "end": END
    }
)

# 工具执行完后，必须回到 agent 汇总信息
workflow.add_edge("tools", "agent")

app = workflow.compile()

# --- 7. 运行测试 ---
print("\n--- 测试：单词长度查询 ---")
for chunk in app.stream({"messages": [HumanMessage(content="单词 'LangGraph' 包含多少个字母?")]}):
    print(chunk)

print("\n--- 测试：时事搜索 ---")
for chunk in app.stream({"messages": [HumanMessage(content="2024年奥斯卡最佳影片是谁?")]}):
    print(chunk)
```

### ToolNode优势

- 代码整洁度：你不再需要写 tool_node 的内部函数逻辑，直接通过 ToolNode(tools) 实例化即可。
- 鲁棒性：它能处理复杂的 tool_calls ID 匹配，确保 LLM 能够准确识别哪个结果对应哪个请求。
- 循环结构 (Loop)： 这是 Agent 模式的核心。图的路径是 Agent -> Tools -> Agent。这种循环允许 Agent 在得到工具结果后，重新审视问题，决定是继续调用其他工具（多步思考），还是直接给出最终答案。
- 状态累加 (operator.add)： 通过在 TypedDict 中使用 Annotated 和 operator.add，所有的对话历史（HumanMessage, AIMessage, ToolMessage）都会被按顺序保存，这为 LLM 提供了完整的决策上下文。
